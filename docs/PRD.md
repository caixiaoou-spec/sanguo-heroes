## 1. 产品定位

**三国英雄**是一款以中国三国时代为背景的**回合制大地图策略游戏**，运行于浏览器（无需安装）。

玩家扮演 14 个历史势力之一（曹操、刘备、孙权等），通过发展城市、招募武将、外交联盟与军事征伐，最终消灭所有对手、统一天下。

**一句话定位**: 轻量化三国志，浏览器里玩的策略征战游戏。

---

## 2. 核心体验目标

| 体验目标 | 说明 |
|----------|------|
| 决策感 | 每回合的行动点有限，玩家必须在内政/外交/军事中做取舍 |
| 历史代入感 | 武将有真实姓名和属性，历史名将有专属技能和决斗台词 |
| 战斗紧张感 | 1v1 顺序决斗，能看到武将出招、血量变化，而非纯数字计算 |
| 扩张满足感 | 城池从少到多、势力从弱到强，有明显的滚雪球正反馈 |

---

## 3. 玩家角色与目标

### 玩家是谁
- 喜欢三国题材的策略游戏爱好者
- 能接受有一定深度但不需要记忆大量操作的玩法
- 在浏览器休闲游玩，单局时长 30~90 分钟

### 玩家的最终目标
消灭所有其他势力（所有城池归己），达成"统一天下"胜利。

### 玩家会失败的情况
己方势力的所有城池被敌人占领，或主将阵亡。

---

## 4. 核心玩法循环

每一回合分为以下阶段，依次执行：

```
① 玩家回合
   └─ 执行最多 3 次行动（内政/外交/行军/分配）
② AI 回合
   └─ 所有其他势力自动执行行动
③ 事件阶段
   └─ 触发随机事件或剧情事件
④ 结算阶段
   └─ 计算金钱/粮草收支，人口增长，行军推进
   └─ 抵达目标城池的行军触发战斗
```

玩家每回合最多执行 **3 个行动**。消耗行动点的操作：
- 城市内政（开发农业/商业、征兵、修缮城防、招贤纳士）各 1 点
- 发起行军 1 点（行军期间消耗 0 点）

不消耗行动点的操作：
- 外交（宣战/停战/结盟/劝降）
- 武将分配士兵
- 查看地图/城市信息

### 5.2 世界地图

**玩家能做什么**
- 在地图上看到所有 55 座城池的位置和归属（颜色区分势力）
- 点击城池，查看城池详情（人口、农业、商业、防御、驻兵、武将列表）
- 查看己方城池与相邻城池的连接关系，判断进攻路线
- 拖拽/滚动地图浏览全局

**地图信息层级**
1. 城池图标：颜色 = 所属势力，大小 = 人口规模
2. 城池名称标签（可缩放显示）
3. 行军路线（箭头动画，显示在途行军）
4. 城池详情面板（点击后展开）

**验收标准**
- [ ] 地图覆盖全部 55 座城池，位置与历史地理大致对应
- [ ] 城池颜色与势力颜色一致，中立城池显示为灰色
- [ ] 点击任意城池弹出详情面板，显示完整城市数据
- [ ] 行军中的部队在地图上有可见动画标识
- [ ] 地图支持拖拽平移和鼠标滚轮缩放

### 5.3 内政系统

**设计意图**: 城市是资源来源，内政决定产出上限。玩家需要在"立即扩张"和"先发育城市"之间做选择。

#### 5.3.1 开发（农业 / 商业）

**玩家能做什么**: 花费金钱提升城市的农业或商业值，增加每回合的粮草或金钱产出。

**操作限制**
- 城中必须有武将（没有武将无法开展内政）
- 需要花费 1000 金
- 消耗 1 行动点
- 农业 / 商业值上限为 100

**玩家能感知到的反馈**
- 操作后城市属性立即更新
- 下回合结算时可看到收入增加

#### 5.3.2 征兵

**玩家能做什么**: 花费金钱为城市补充驻兵，再通过"分配"操作将士兵交给城中武将。

**操作限制**
- 需要花费 2000 金
- 征兵后城市士气略降（-3）
- 消耗 1 行动点
- 征兵上限：该城人口的 2%

#### 5.3.3 修缮城防

**玩家能做什么**: 花费金钱提升城市防御值，使守城战更难被攻破。

**操作限制**
- 需要花费 1500 金
- 消耗 1 行动点
- 防御值上限为 100

#### 5.3.4 招贤纳士

**玩家能做什么**: 派遣城中武将去招募在野人才，有概率获得新武将加入己方。

**操作限制**
- 城中必须有武将（以魅力属性影响成功率）
- 消耗 1 行动点
- 招募到的武将初始忠诚度较低（60~80），需要时间培养

#### 5.3.5 分配士兵

**玩家能做什么**: 将城市驻兵分配给城中武将，使武将在出征时携带兵力。

**操作限制**
- 每名武将携兵上限 = 统帅值 × 50
- 不消耗行动点

**验收标准**
- [ ] 金钱不足时，所有内政操作显示禁用状态，并提示原因
- [ ] 城中无武将时，开发和招贤显示禁用状态
- [ ] 所有内政操作执行后，数值实时更新并有操作反馈提示
- [ ] 农业/商业/防御值达到 100 时，对应操作显示"已达上限"

### 5.4 外交系统

**设计意图**: 玩家不能同时对所有势力开战，外交是控制战线宽度的工具。

#### 宣战
- 向目标势力宣战，解锁对该势力城池发起进攻
- 宣战后双方自动进入敌对状态，原同盟关系解除
- 已处于交战状态时无法重复宣战

#### 停战
- 花费 5000 金尝试与交战方停战
- 对方接受概率受己方名声影响（名声越高接受率越高）
- 对方可能拒绝，金钱照扣

#### 结盟
- 向目标势力提出结盟，结盟后双方不可互相攻击
- 对方接受概率受双方名声、势力强弱对比影响
- 交战中的势力不可结盟

#### 劝降
- 当目标势力只剩 1~2 座城池时，可尝试劝其投降
- 成功后目标势力灭亡，其城池和武将并入己方（武将忠诚度较低）
- 成功概率受双方名声差影响

**关系状态说明**
| 关系 | 可否攻击 | 说明 |
|------|----------|------|
| 中立 | 可 | 默认状态，可随时发起进攻（但名声会受影响） |
| 同盟 | 否 | 结盟后受保护，违约宣战名声大幅下降 |
| 交战 | 可 | 宣战后进入此状态 |

**验收标准**
- [ ] 外交面板显示当前与所有势力的关系状态
- [ ] 已是同盟的势力，"宣战"按钮有二次确认弹窗（"违约宣战将损失名声"）
- [ ] 已处于交战状态时，"宣战"按钮置灰
- [ ] 停战/结盟/劝降操作有明确的成功/失败提示，失败时说明原因
- [ ] 劝降选项只在目标势力 ≤ 2 城时才显示可用

### 5.5 行军系统

**设计意图**: 距离和时间让玩家感受到空间感，攻城不是即时的，需要提前规划。

**行军类型**: `'attack'`（进攻敌方/中立城池）| `'transfer'`（己方城池间调遣武将）

**玩家能做什么**
- 选择一座己方城池和一名（或多名）武将，向相邻或更远的城池发起行军
- 根据距离，行军需要 1~4 个回合才能抵达
- 行军中的武将不能执行其他操作
- 抵达目标城池后自动触发战斗（如果目标是敌方或中立城池）

**行军时间**
| 距离 | 所需回合 |
|------|----------|
| 近（≤60） | 1 回合 |
| 中（≤120） | 2 回合 |
| 远（≤200） | 3 回合 |
| 极远（>200） | 4 回合 |

**行军期间的显示**
- 地图上出现行军箭头，标注剩余回合数
- 参与行军的武将在城市详情中显示"行军中"状态

**验收标准**
- [ ] 发起行军时，地图上立即出现带箭头的行军路线标识
- [ ] 行军对象在城市武将列表中标注"行军中"，不可再次选中
- [ ] 每回合结算后，行军剩余回合数 -1，地图上实时更新
- [ ] 抵达目标城池后自动进入战斗流程或占领（若无守兵）
- [ ] 取消行军的武将返回出发城池

### 5.6 战斗系统

**设计意图**: 战斗是策略的最终表达，1v1 顺序决斗让每名武将都有存在感，而不是兵力碾压的数字游戏。

#### 战斗发起条件
- 进攻方行军部队抵达目标城池
- 目标城池有守军（否则直接占领）

#### 战斗流程

```
双方各出最多 3 名武将
  → 依次进行 1v1 决斗（顺序决斗制）
  → 每场决斗最长 60 秒
  → 积分制：胜一场得 1 分
  → 3 场全胜，或对方武将全部阵亡，战斗结束
```

#### 玩家在战斗中能做什么
- 选择己方武将出战顺序
- 手动触发武将技能（对手 AI 自动使用技能）
- 选择出战阵型（影响战斗加成）

#### 阵型效果
| 阵型 | 效果 | 适合场景 |
|------|------|----------|
| 圆形阵 | 开局 HP +10% | 防守/持久战 |
| 锋矢阵 | 攻击伤害 +10% | 快攻/以强打弱 |
| 鱼鳞阵 | 受到伤害 -15% | 硬撑/以弱打强 |

#### 兵种克制关系
```
骑兵 → 步兵（1.2×）
枪兵 → 骑兵（1.5×）
步兵 → 弓兵（1.3×）
弓兵 → 骑兵（1.2×）
```

#### 历史决斗台词系统

53 组历史名将决斗，每对决斗武将有专属对话和背景故事。例如：吕布 vs 关羽、赵云 vs 张郃、诸葛亮 vs 司马懿等，决斗开场和结束时触发专属台词与历史典故说明。

#### 战斗结果
- **进攻方胜**: 占领城池，防守方武将有概率被俘或撤退
- **防守方胜**: 进攻方撤回出发城池，损失兵力
- **双方武将均获得经验值**，可能触发升级

**验收标准**
- [ ] 进攻方抵达目标城池后，自动进入战斗准备界面，玩家可选择参战武将（最多 3 名）
- [ ] 每场 1v1 决斗中，玩家可点击技能按钮手动释放技能
- [ ] 技能有 MP 消耗和冷却，MP 不足或冷却中按钮显示禁用
- [ ] 决斗结束后，清晰显示胜负和当前积分（如 2:1）
- [ ] 战斗结束后，结果界面显示：城池是否被占领、武将状态（存活/被俘/撤退）、经验获得
- [ ] 兵种克制关系在选将界面有视觉提示

### 5.7 武将系统

**设计意图**: 武将是玩家在游戏中最有感情投入的对象，成长系统让玩家有理由保护他们。

#### 武将属性
| 属性 | 作用 |
|------|------|
| 武力 | 决定战斗伤害和 HP |
| 智力 | 决定技能伤害和 MP 上限 |
| 统帅 | 决定携兵上限和防御力 |
| 政治 | 影响内政开发效果 |
| 魅力 | 影响招贤成功率和外交概率 |

#### 武将成长
- 参与战斗获得经验值
- 经验积累到阈值后自动升级（阈值 = 当前等级 × 100）
- 升级后 HP/MP 上限提升，并可能解锁新技能
- 等级上限 50 级

#### 技能系统
- 每名武将有 2~3 个技能
- 技能分类：武将技（高伤害）、军师技（范围/特效）、辅助技（回血/增益）
- 历史名将（如项羽）有专属独特技能

#### 武将状态
| 状态 | 说明 |
|------|------|
| 正常 | 可正常使用 |
| 行军中 | 正在行军，不可分配其他任务 |
| 阵亡 | 战斗中 HP 归零，暂时无法使用 |
| 被俘 | 被敌方俘虏，需赎回或等待 |

**验收标准**
- [ ] 武将详情页显示所有属性、当前等级/经验进度、技能列表
- [ ] 升级时有明显的视觉/音效反馈，并显示属性变化数值
- [ ] 技能列表显示每个技能的冷却状态和 MP 消耗
- [ ] 武将处于"行军中"或"被俘"状态时，在列表中有明显标识

---

### 5.8 事件系统

**设计意图**: 随机事件打破玩家的计划，历史剧情事件制造代入感，两者共同增加游戏的不确定性和重玩价值。

#### 事件类型

**随机事件**（每回合 30% 概率触发）
- 针对玩家随机一座城池发生
- 示例：丰收（农业+10）、瘟疫（人口-5%）、火灾（士气-15）、商队来访（金钱+2000）

**剧情事件**（条件触发，每个只触发一次）
- 满足特定条件（回合数、名声、城池数等）后触发
- 通常有多个选项供玩家选择，不同选项产生不同结果
- 示例：官渡之战前的外交选择、赤壁之战时的联盟机会

#### 事件效果类型
- 资源变化：金钱、粮草增减
- 城市变化：人口、农业、商业、士气增减
- 武将变化：忠诚度增减、新武将加入
- 名声变化：势力名声增减

**验收标准**
- [ ] 事件触发时弹出事件卡片，包含事件名称、描述文本、插图（可选）
- [ ] 有多个选项的事件，每个选项显示可能的效果提示（不透露具体数值，仅方向）
- [ ] 玩家做出选择后，效果立即生效并在通知栏显示结果
- [ ] 每个剧情事件在整局游戏中只触发一次（重置后可再次触发）
- [ ] 随机事件不重复显示相同事件（同一回合）

---

### 5.9 胜负判定

#### 胜利条件
消灭所有其他势力（所有城池归己方所有）→ 显示"统一天下"胜利画面

#### 失败条件
己方势力的最后一座城池被敌人占领 → 显示"兵败身亡"失败画面

#### 势力灭亡判定
当一个势力失去所有城池，该势力标记为"已灭亡"，其剩余武将（如有）流散为在野武将，可被其他势力招募。

**验收标准**
- [ ] 胜利/失败时，游戏立即暂停并弹出结果画面
- [ ] 结果画面显示：本局回合数、统一城池数、击败势力列表
- [ ] 结果画面提供"重新开始"和"返回主菜单"两个选项
- [ ] 势力灭亡时，在通知栏显示"XX 势力已灭亡"消息

## 6. 数值设计意图

这一章说明关键数值的**设计理由**，方便后续调整时保持平衡意图。

### 6.1 行动点（3 点/回合）
设计为 3 点是为了让玩家每回合面临取舍：一般情况只能选 3 件事做，不能既大量征兵又大量开发。过多行动点会消除决策压力。

### 6.2 行军距离阈值
4 个档位（1~4 回合）对应地图上的近/中/远/极远，让玩家有直觉感受。跨越整张地图最长需要 4 回合，保证防守方有时间响应。

### 6.3 兵种克制倍率（1.2~1.5×）
倍率设计为"有优势但不是碾压"——枪兵克骑兵 1.5× 是最高倍率，意味着同等属性下枪兵明显胜，但武力极高的骑兵将（如吕布）仍可逆克。

### 6.4 停战金钱（5000 金）
5000 金约等于中型城市 3~4 回合的商业收入，设计为"有代价但早期可负担"。避免停战代价过低让外交失去意义。

### 6.5 武将携兵上限（统帅 × 50）
统帅 80 的将领最多带 4000 士兵，统帅 30 的谋士最多带 1500。设计意图是让高统帅武将在大规模战役中有明显优势，同时谋士也有参战价值（依靠技能而非士兵）。

### 6.6 升级经验阈值（等级 × 100）
1 级升 2 级需 100 经验，50 级升级需 5000 经验。前期升级快、后期趋缓，保证早期武将成长有明显感受，后期不会过快堆满。

## 7. 用户故事与验收标准

### US-01 选择势力
**作为**玩家，**我希望**在游戏开始时选择一个自己喜欢的历史势力，**以便**以该势力的视角体验游戏。

**验收标准**
- 展示全部 14 个势力，每个势力有名称、代表武将、起始实力简介
- 选择后可查看势力详情再确认
- 确认后进入游戏，地图高亮显示己方城池

### US-02 规划发展路线
**作为**玩家，**我希望**通过内政提升城市产出，**以便**积累资源支撑后续扩张。

**验收标准**
- 每个城市的农业/商业/防御当前值和上限清晰可见
- 内政操作前可预览操作效果（如"开发农业：+8，花费 1000 金"）
- 操作后收入变化在下回合结算时有对比显示

### US-03 发动战争
**作为**玩家，**我希望**向相邻势力的城池发起进攻，**以便**扩大领土。

**验收标准**
- 选择己方城池、目标城池、参战武将三步完成发兵
- 地图上清晰显示行军进度和预计抵达回合
- 战斗结果（胜/败、城池归属变化）有明确通知

### US-04 观看决斗
**作为**玩家，**我希望**在战斗中看到武将之间的精彩决斗过程，**以便**感受武将的个性和强度。

**验收标准**
- 决斗画面展示双方武将形象、血量条、MP 条
- 普通攻击有动作动画和打击特效
- 技能释放有独特视觉效果（不同技能类型视效不同）
- 血量变化与动画在视觉上同步（不是即时扣血）

### US-05 使用外交手段
**作为**玩家，**我希望**与弱小势力结盟以避免两线作战，**以便**专注进攻主要对手。

**验收标准**
- 外交界面一览当前与所有势力的关系状态
- 结盟/停战操作有明确的成功率提示（"高/中/低"而非具体数值）
- 同盟关系在地图上有视觉标识（城池边框或颜色区分）

## 8. 超出范围（Not in Scope）

以下功能**当前版本不包含**，后续版本可考虑：

| 功能 | 说明 |
|------|------|
| 多人联机 | 当前为单人游戏，AI 控制所有非玩家势力 |
| 存档云同步 | 当前存档保存在本地 localStorage |
| 移动端适配 | 当前针对桌面浏览器设计，未针对触屏优化 |
| 剧情动画/语音 | 武将无配音，事件无动画过场 |
| 自定义地图 | 地图为固定历史地图，不支持自定义 |
| 武将装备系统 | 当前无装备槽位，武将属性固定（通过升级提升） |
| 粮草运输机制 | 粮草直接全局结算，无运输线/补给线机制 |
